# 记账本 UI 风格与参考图设计说明

本文档根据项目沟通过程中提供的**多张参考界面图**，整理画面中的**元素、风格语言、色彩与质感要求**，并说明**本应用（记账本）应落到的标准**（以**金融类深色稿**为主，兼收其它参考中的通用原则）。  
色值与实现以 `lib/core/theme/app_colors.dart` 为准，便于设计与开发对照。

---

## 1. 参考图族与共同特征

你提供的图片可粗分为几类（以下用「风格族」描述，不绑定具体产品）：

| 风格族 | 画面特征（从图中抽象） | 与记账本的关系 |
|--------|------------------------|----------------|
| **金融 / 银行类** | 近黑底、略亮一级的卡片、铜/金暖色氛围光、柔和蓝作为链接与选中、顶栏头像+铃铛+搜索、底部磨砂条 | **主参考**：首页顶栏、底栏、整体深色层级 |
| **探索 / 助手类** | 大圆角卡片网格、彩色图标、玻璃感圆形主按钮、背景弥散光斑 | 借鉴：**圆角尺度、卡片分组、不要用杂乱描边** |
| **智能家居类** | 紫蓝/薄荷绿渐变、大圆环控件、彩虹色环、金属旋钮 | **仅借鉴色调对比度**；**不要求**实现圆环、色盘等复杂控件 |
| **加密货币 / 行情类** | 霓虹绿/红、K 线、高密度数据卡片 | 借鉴：**涨跌色（绿/红）、深色卡片上的数据可读性**；**不要求**定制 K 线炫技 |

**记账本的约定**：整体气质对齐 **Premium Dark + 金融应用**，强调**可读性、层级清晰、柔和氛围光**，避免「炫技型」自定义绘制（大转盘、色轮、复杂图表皮肤等），仅在主题与通用组件层面统一风格。

## 2. 核心设计关键词（必须对齐）

- **深色优先**：主界面以深色模式为视觉基准；浅色模式保持简洁即可。
- **近黑背景**：页面底色接近 OLED 黑，但不是刺眼的 `#000000` 纯黑一片；允许极弱渐变或光晕打破沉闷。
- **卡片抬升一级**：内容区块背景比页面背景**略亮、略暖或略灰**，形成层级；少用硬边框，可用**极低不透明度的浅色描边**模拟玻璃边缘。
- **暖色氛围（铜/桃）**：用于**顶部区域、问候或大标题点缀**，面积小、透明度低，避免抢戏。
- **冷色交互（蓝/青）**：用于**主按钮、链接、Tab 选中、底部导航选中**，与暖氛围形成对比。
- **线性图标**：导航与工具区优先 **Outlined / 线框** 图标；填充图标可用于快捷入口内部，但保持风格统一。
- **大圆角**：卡片与主要容器圆角 **约 20～28px**（按逻辑像素）；按钮、输入框略小一档 **约 12～16px**。
- **柔和阴影**：深色模式下阴影偏**扩散、低不透明**，避免生硬投影。

## 3. 色彩系统（详细）

### 3.1 深色模式 — 页面结构色

| 用途 | 说明 | 参考色值（已实现） |
|------|------|-------------------|
| **页面背景（Scaffold）** | 极深灰黑，略好于纯黑 | `#0D0D0D` `darkBackground` |
| **卡片 / 一级表面** | 比背景亮一档的灰，类似 iOS 深色卡片 | `#1C1C1E` `darkSurface` |
| **二级表面 / 输入底** | 再亮一点，用于嵌套块、输入框填充 | `#2C2C2E` `darkSurfaceVariant` |
| **主文字** | 高对比白 | `#FFFFFF` `darkOnSurface` |
| **次级文字 / 说明** | 中灰，保证 WCAG 可读前提下偏柔和 | `#9E9E9E` `darkOnSurfaceVariant` |
| **分割线 / 弱边界** | 低对比轮廓 | `#38383A` `darkOutline` |
| **卡片轻描边（可选）** | 白色约 **8%** 透明度叠在边缘，增强玻璃感 | `darkCardBorder`（`#14FFFFFF` 量级） |

### 3.2 交互与品牌色（冷侧）

| 用途 | 说明 | 参考色值（已实现） |
|------|------|-------------------|
| **主色 Primary** | 柔和天蓝，用于主按钮、强调链接、底栏选中图标与文字 | `#5EB8FF` |
| **主色偏亮** | hover / 高亮态（若需要） | `#82C9FF` `primaryLight` |
| **主色偏暗** | 按下或次要强调 | `#3D9EE9` `primaryDark` |
| **次要色 Secondary** | 青色系，用于渐变第二色、辅助图标 | `#26C6DA` |
| **次要色偏亮** | 辅助高光 | `#4DD0E1` `secondaryLight` |

### 3.3 氛围与装饰色（暖侧）

| 用途 | 说明 | 参考色值（已实现） |
|------|------|-------------------|
| **铜色 Accent** | 顶部光晕、头像外圈光、径向氛围 | `#D4A574` `accentCopper` |
| **桃色 Accent** | 可与铜色一起做渐变，用于标题渐变字（若使用） | `#E8C4A8` `accentPeach` |

参考图中的 **「人像姓名上的铜桃渐变」**、**「顶部大面积铜色弥散光」** 均属于此类；实现时应用 **低不透明度叠层**，避免整块不透明暖色。

### 3.4 功能与财务语义色

| 语义 | 参考色值（已实现） |
|------|-------------------|
| 成功 / 收入 / 正向 | `#00E676` |
| 错误 / 支出（强调） | `#FF5252` |
| 警告 | `#FFB020` |
| 信息 | `#448AFF` |
| 通知红点（参考图中铃铛上的数字） | 饱和度足够的红，与错误色同系即可 `#FF5252` |

加密类参考图中的 **霓虹绿 / 珊瑚红** 与上述方向一致：在深色底上略提高饱和度，便于扫读。

### 3.5 主摘要卡片渐变（结余区等）

参考金融类深色 Banner：**深色底上的内敛渐变**，而非高饱和紫粉大块。当前实现为：

| 起点（左上偏暖褐） | 终点（右下偏深蓝黑） |
|-------------------|---------------------|
| `#2A2420` `balanceGradientStart` | `#121826` `balanceGradientEnd` |

视觉上：**暖褐 → 冷深蓝黑**，与顶部铜氛围呼应且不花哨。

### 3.6 浅色模式（次要）

浅色用于系统强制浅色或用户切换：背景浅灰白、表面纯白、主字深蓝灰、次级灰字；主色仍使用同一套 **蓝青** 以保持品牌一致。详见 `app_colors.dart` 中 `light*` 常量。

## 4. 参考图中的界面元素拆解（按区域）

以下按**你在图中能看到的结构**描述，并对应到「记账本应做到的样子」。

### 4.1 顶部区域（金融类参考为主）

- **背景**：深色统一底上，常见 **从屏幕下方中央向上扩散的铜色/暖色径向光**，像一盏柔光灯打在顶栏下半部分；亦可叠加极弱的纵向渐变，使顶部略暖、向下融入背景。
- **左侧**：**圆形头像占位** — 浅灰底、深色剪影人形图标、**细白内描边**；外圈可有 **铜色柔光阴影**（非实心粗环）。
- **右侧**：**线框铃铛** + **角标红点（白字数字）**；旁为 **线框放大镜（搜索）**。图标颜色在深色模式下为 **纯白或近白**。
- **标题**：参考图常 **弱化或取消** 大面积居中标题，避免与头像、工具按钮抢视觉；若保留标题，可用 **铜→白渐变字** 或小号字，而非大块纯色标题。

**记账本落地**：深色首页顶栏采用 **底缘铜色弧光 + 左头像 + 右铃铛与搜索**；不强塞大号「记账本」标题（浅色模式可保留标题）。

### 4.2 卡片与列表块

- **外形**：圆角大，**20～28px** 量级；成组信息放在同一卡片内，用 **细分隔线**（深灰、低对比）区分行。
- **颜色**：卡片背景 **`#1C1C1E` 一级表面**；内再分层可用 **`#2C2C2E`**。
- **边缘**：可用 **1px、约 8% 白透明** 描边 + **软阴影**，做出轻微「浮起」感；不必做重度毛玻璃（除非单独封装且性能允许）。

### 4.3 快捷入口（方形格子）

参考图中常见 **深色方块 + 线框白色图标**，而非彩虹底每个格子不同高饱和底色。  
**记账本落地**：深色模式下快捷入口为 **深灰表面 + 细描边 + 近白图标**；浅色模式可保留淡色浅底 +  Brand 色图标。

### 4.4 分段 / Tab / 筛选胶囊

- **未选中**：透明或融入背景，文字 **次级灰**。
- **选中**：**浅灰药丸底（表面变体色）+ 主文字白或主色**，或 **下划线 / 品牌色文字**（依组件而定）。
- **边框**：整体可用 **细轮廓色** `#38383A` 圈住一组分段按钮。

### 4.5 底部导航

参考 **磨砂玻璃条**：**半透明深色底 + 背景模糊**；顶边可有 **极细亮线**（约 6% 白）。  
**选中项**：**蓝色图标 + 蓝色标签**（与主色一致），无需白色圆形衬底（除非产品明确要求）。

中间 **主行动按钮（如记账 +）**：**圆形**，填充 **蓝→青渐变**，带 **柔和外发光阴影**（主色半透明）。

### 4.6 图表与数据（统计页）

- 折线/柱状：**品牌蓝青 / 绿红语义色**，在深色卡片上对比足够即可。
- **不要求**与加密货币客户端一致的 **K 线精细渲染、十字光标炫技**，除非产品单独排期。

---

## 5. 字体与排版层级

- **字体家族**：中文 **Noto Sans SC**（项目已通过 `google_fonts` 使用），英文可读性可与 Inter / SF 类似几何无衬线协调。
- **层级建议**：
  - 页面标题 / 余额大数字：**Bold / W600**，主白。
  - 区块标题：**W600**，主白。
  - 辅助说明、列表次要信息：**Regular**，**次级灰** `#9E9E9E`。
  - Tab / 底部导航标签：**小字号**，选中可加粗。

---

## 6. 动效与质感（克制）

- **模糊**：用于 **底部导航、少量浮层**，`backdrop-filter` 类效果；避免全屏滥用导致性能问题。
- **渐变**：**线性渐变**用于按钮、Banner；**径向渐变**用于顶部氛围光；透明度控制在 **6%～45%** 区间为宜。
- **炫技控件**：**圆形温控盘、彩虹色环、金属旋钮、定制 K 线引擎** 等 — **不作为记账本必选**，仅在风格文档中标注为「参考图元素，本产品不实现」。

## 8. 验收时可对照的检查清单

- [ ] 深色背景是否为 **`#0D0D0D`** 量级，而非发灰发蓝的大平板。
- [ ] 卡片是否为 **`#1C1C1E`**，且与背景对比清晰但不刺眼。
- [ ] 主交互是否为 **柔和蓝**，而非旧版大紫大粉。
- [ ] 顶栏是否有 **底部中央铜色柔光**（若首页已按金融稿实现）。
- [ ] 顶栏是否为 **左头像、右铃铛+搜索**，图标为 **线框、浅色**。
- [ ] 底栏是否 **半透明 + 模糊**，选中是否为 **蓝色图标与文字**。
- [ ] 收入/支出是否分别为 **绿 / 红**，且饱和度在深色上可读。
- [ ] 快捷入口深色是否为 **深块 + 白图标**，避免五色缤纷小方块。
