# 记账 APP 需求文档

## 项目概述

**项目名称**: 记账本 (Bookkeeper)
**版本**: 1.9.0
**平台**: Android (Flutter 跨平台)
**更新日期**: 2026-05-13

---

## 1. 项目背景

用户需要一个功能完整、界面美观的记账 APP，用于日常收支管理。要求：
- 界面风格采用 Peekaboo 柔和梦幻设计（梦幻紫 `#B8A9E8` + 暖黄强调 `#FFD93D`）
- 支持高级记账功能（多币种、标签、周期记账、数据导出）
- 支持微信/支付宝账单导入
- 支持支付通知监听自动记账

---

## 2. 功能需求

### 2.1 核心记账功能

#### 2.1.1 收支记录
- **输入项**: 金额（必填）、分类（必填）、账户（必填）、日期（默认当天）、商品、备注、标签（多选）、图片附件
- **操作**: 新增/编辑/删除/批量删除、搜索筛选、按日期分组
- **计算器**: 支持加减法连续计算

#### 2.1.2 分类管理
- 系统内置默认分类（不可删除）：10 个支出 + 4 个收入
- 用户自定义分类（图标/颜色可选）
- 分类排序（拖拽手柄）
- **关联贷款**：支出分类可关联贷款账户，记账时自动扣减余额

#### 2.1.3 账户管理
- 默认账户：现金、银行卡、支付宝、微信
- 账户余额自动计算
- 贷款账户单独管理（type=loan）

### 2.2 高级功能

#### 2.2.1 标签系统
- 标签 CRUD + 交易多选标签 + 按标签筛选

#### 2.2.2 多币种支持
- 8 种货币（CNY、USD、EUR、GBP、JPY、KRW、HKD、TWD）
- 自动获取汇率 + 24 小时缓存

#### 2.2.3 周期记账
- 支持频率：每分钟/每小时/每日/每周/每月/每年
- 后台定时检查、自动生成交易记录、执行后通知

#### 2.2.4 预算管理
- 月度总预算 + 分类预算
- 进度显示 + 超支提醒

#### 2.2.5 统计报表
- 饼图（分类占比）、折线图（收支趋势）、柱状图（分类排行）
- 时间维度：本周/本月/本季/本年/自定义
- 月份切换器

#### 2.2.6 数据导出
- CSV / Excel（.xlsx）格式
- 选择时间范围和导出字段、分享文件

### 2.3 自动化功能

#### 2.3.1 账单导入
- 来源：微信支付（CSV/XLSX UTF-8）、支付宝（CSV/XLSX GBK）
- 重复记录检测、自动分类匹配、导入预览

#### 2.3.2 智能记账（支付通知监听）
- Android NotificationListenerService 监听微信/支付宝/银行通知
- 正则解析金额、商户名、收支方向
- 去重机制（10秒窗口）
- 前台：MethodChannel → 确认弹窗
- 后台：SQLite pending_payments → 系统通知 → 待确认列表

#### 2.3.3 个人中心 + 贷款管理
- 用户档案（头像、等级、头衔、经验值）
- 荣誉室（5 个成就徽章）
- 资产摘要（总资产/净资产/负债实时计算）
- 贷款管理（房贷/车贷/信用贷/网贷/其他）
- 分类关联贷款 + 记账自动还款扣减

## 3. 非功能需求

### 3.1 性能
- 应用启动 < 2秒
- 列表滚动 60fps
- 数据库查询 < 100ms

### 3.2 安全
- 本地数据存储（无云端泄露风险）
- 支持数据备份/恢复

### 3.3 用户体验
- Peekaboo 柔和梦幻视觉风格
- 深色/浅色双模式 + 跟随系统
- 中文界面 + 流畅动画（flutter_animate）
- `PageRouteBuilder` + `FadeTransition` 页面切换（无闪烁）

### 3.4 兼容性
- Android 5.0 (API 21) 及以上
- 支持主流屏幕尺寸

## 4. 技术架构

### 4.1 技术栈

| 组件 | 选择 | 理由 |
|------|------|------|
| 框架 | Flutter | 跨平台 |
| 状态管理 | Riverpod | 编译时安全 |
| 数据库 | sqflite (SQLite) | 原生查询 |
| 路由 | go_router | 声明式 |
| 图表 | fl_chart | 饼图/折线/柱状图 |
| 动画 | flutter_animate | 链式声明式动画 |
| 通知监听 | Android NotificationListenerService | 原生服务 |

### 4.2 架构模式
- Feature-first + Clean Architecture
- Presentation / Domain / Data 分层
- 响应式编程（Stream + Riverpod）
