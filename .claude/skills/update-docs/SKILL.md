# 文档自动更新

**核心规则：每次修改代码后，必须自动更新所有相关文档，无需用户提醒。**

## 触发时机

以下情况必须自动执行文档更新：
- 完成任何代码修改后
- 修复 bug 后
- 实现新功能后
- 优化现有功能后
- 用户指出文档未更新时

## 更新内容

### 1. 必更新：docs/changes.md
每次代码变更都要添加记录，格式：
```markdown
### N. 变更标题
**时间**: YYYY-MM-DD HH:mm:ss
**原因**: 为什么要修改
**文件**: 修改了哪些文件
**变更**:
- 具体修改内容
```

### 2. 条件更新

| 文档 | 触发条件 |
|------|----------|
| `docs/bugs.md` | 涉及 bug 修复时，更新状态为已修复 |
| `docs/features.md` | 涉及新功能时，更新功能列表 |
| `docs/changelog.md` | 版本发布时，添加版本记录 |
| `docs/requirements.md` | 需求变更时，更新需求文档 |

## 版本号规则

采用语义化版本：
- **修订号 (PATCH)**: bug 修复、小优化
- **次版本号 (MINOR)**: 新功能
- **主版本号 (MAJOR)**: 不兼容的重大变更

## 文档规范

- 使用中文
- changes.md 按日期倒序（最新在前）
- 时间精确到秒
- 保持格式一致
